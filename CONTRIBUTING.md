## Getting Started

1. Fork the repository
2. Clone your fork
3. Create a feature branch: `git checkout -b feature/amazing-feature`
4. Make your changes
5. Commit: `git commit -m 'feat: Add amazing feature'`
6. Push: `git push origin feature/amazing-feature`
7. Open a Pull Request

## Development Setup

```bash
git clone https://github.com/YOUR-USERNAME/free-games-claimer.git
cd free-games-claimer

# Backend
cd backend
npm install
npm run dev

# Frontend (new terminal)
cd frontend
npm install
npm run dev
```

## Code Style

- Use ES6+ syntax
- Use meaningful variable names
- Add comments for complex logic
- Keep functions small and focused
- Follow existing code patterns

## Commit Messages

```
feat: Add new feature
fix: Fix bug
docs: Update documentation
style: Format code
refactor: Restructure code
test: Add tests
chore: Update dependencies
```

## Testing

```bash
# Backend tests
npm test

# Frontend tests (coming soon)
npm test
```

## Pull Request Process

1. Update README.md with any new features
2. Update .env.example with new environment variables
3. Test your changes thoroughly
4. Ensure code follows style guide
5. Request review from maintainers

## Reporting Issues

1. Check if issue already exists
2. Provide clear description
3. Include steps to reproduce
4. Include error logs
5. Include system information

## Feature Requests

1. Describe the feature clearly
2. Explain why it's needed
3. Provide examples if possible
